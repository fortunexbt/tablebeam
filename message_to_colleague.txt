I've just pushed a new branch called `cloud-infrastructure` to the client-tracker-assistant repo. This is a separate branch from `main` that adds full cloud deployment capabilities.

## What's New

The `cloud-infrastructure` branch includes:
- Dockerized app ready for Kubernetes with GPU support (A100s)
- REST API endpoint so the team can just hit an API instead of running Ollama locally
- Complete K8s manifests with auto-scaling
- ArgoCD setup for GitOps deployment
- GitHub Actions CI/CD to push images to GHCR
- Pinecone integration as a cloud alternative to ChromaDB
- Shared model cache volume to keep cold starts fast

## Important Note

This is intentionally a separate branch, not a PR to main, because:
- `main` branch = local development (current setup)
- `cloud-infrastructure` branch = production K8s deployment

They serve different purposes and audiences.

## Next Steps for Deployment

1. Set up secrets in your K8s cluster:
   ```bash
   kubectl create secret generic client-tracker-secrets \
     --from-literal=pinecone-api-key=YOUR_KEY \
     -n client-tracker
   ```

2. Deploy with ArgoCD:
   ```bash
   kubectl apply -f argocd/application.yaml
   ```

3. The API will be available at the LoadBalancer endpoint

Check out `DEPLOYMENT.md` in the branch for the full guide.
